# O problema do Hello World

Acredito que tudo que falei foi bem óbvio, mas por que quis trazer novamente esse nível de consciência? Porque sabemos que, quando estamos criando uma estrutura básica de uma aplicação, ao rodarmos um comando para criar, por exemplo, um *Next App*, ele vai gerar uma série de arquivos. Mas, na prática, existem duas situações.

Se você trabalha em qualquer empresa, **raramente essa estrutura será seguida para o restante do projeto**. A estrutura base que esses comandos trazem ajuda inicialmente, mas tem uma tendência muito grande de mudar. Não há nada de errado com isso. Obviamente, alguns frameworks são mais opinativos, com regras claras sobre onde os arquivos devem ficar, mas mesmo sendo opinativos, ainda faremos customizações nas estruturas. Você pode precisar trabalhar com testes de um jeito específico, com outro tipo de arquitetura, com *Clean Architecture*, de forma completamente diferente. Essas mudanças são naturais.

Não é porque gerei um comando e ele produziu uma estrutura base que essa estrutura vai resolver minha vida. Sei que ao longo do tempo ela vai mudar, e muitas vezes já sabemos o que vai mudar. Muitas vezes já temos até um **template** pronto no GitHub, onde damos um `clone` para não começar do zero.

Agora, quando trabalhamos com IA para começar o projeto, entramos em outro aspecto. Comecei meu projeto. O que vou precisar? Vou precisar de um agente, de skills. O que mais preciso? Preciso trabalhar com testes? Preciso de scripts de inicialização para que meu agente consiga subir e descer rapidamente a aplicação, para verificar logs, depurar, etc. Existem diversas etapas até eu fazer, entre aspas, meu **Hello World**.

Lembro que quando trabalhava há muito tempo atrás, fazíamos um *Hello World* gerando um arquivo simples. Depois, com um framework, se digitava um comando que gerava 100 arquivos e saía o mesmo *Hello World*. A grande diferença é que o *Hello World* era apenas a ponta do *iceberg*. No meio, havia todo um preparo e complexidade que realmente importavam.

Quando trabalhamos com IA, **as coisas voltaram a acontecer dessa forma**. Quando gero agora meu *Hello World* com esses scripts, ele traz aquela fundação antiga, mas agora temos outra fundação em paralelo que precisamos ter. Essa fundação é onde a IA atua. **Não consigo trabalhar decentemente com IA se não tenho a infraestrutura base onde meu software vai ser desenvolvido.**

Tenho que ter, por exemplo:

- As **regras claras de um agente**
- Os **skills** que vou utilizar
- Como vou realizar os **testes** e as regras para fazer os testes
- Aqueles **scripts de inicialização** para subir e descer a aplicação, verificar se está funcionando, qual porta está utilizando
- Conseguir visualizar **logs**
- Ter formas para que a IA **navegue no browser** e faça testes de UI

Há tanta preparação de infraestrutura para que esse *Hello World* gerado com IA também funcione.

Esse é o nosso ponto principal. **A maior dificuldade de todo mundo hoje em dia é: qual é o meu Hello World com IA?** A partir de onde posso começar o meu projeto para saber que tudo que fizer a partir de agora, a IA realmente vai poder me ajudar, para eu não ficar batendo cabeça?

Falamos muito sobre esses aspectos: Skills, arquivos de inicialização, etc. Mas o que é o mínimo necessário? Uma coisa tenho certeza: se você criar o seu projeto com aquele comando que vai funcionar, e começar a usar IA sem esse bloco inicial, você simplesmente vai demorar, vai ter uma curva enorme até ela começar a ser produtiva para você, porque você não está trazendo a fundação base para ela.

Nosso objetivo é trazer essa fundação base e, ao mesmo tempo, trazer essa estrutura. **Nossa ideia é juntar esses dois mundos**, para que o meu *Hello World* saia utilizando todos aqueles milhares de arquivos que o framework gera, mas que também saia com todos os arquivos que preciso para que a IA consiga trabalhar a partir daquele momento.

É isso que vejo as pessoas perdendo mais tempo. Elas vão por um caminho, e nós vamos de forma progressiva, começando a gastar tempo e *token* porque não geraram essa estrutura mais personalizada desde o início.
