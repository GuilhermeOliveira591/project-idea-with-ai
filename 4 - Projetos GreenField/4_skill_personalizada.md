# Skill personalizada

No vídeo anterior falei sobre o script que gera as coisas da forma como estamos acostumados, mas precisamos de um *hello world* da IA também para ela começar a trabalhar no **dia zero**. Como recomendo que você faça? Não estou dizendo que é a única forma, mas é uma forma que tenho trabalhado e acredito que está funcionando muito bem.

Você pode **criar uma skill customizada** para que, na hora que pedir para gerar sua aplicação, ela já saia preparada para você trabalhar com IA. Ou seja, opte por uma skill personalizada para fazer o ***bootstrap*** do seu projeto. Evite fazer com que a IA fique gerando arquivos manualmente. **Utilize scripts de geração.** Esses scripts podem clonar, inclusive, um repositório.

Se você não fica mais rodando aqueles comandos de Next.js, se tem um repositório da empresa com um *boilerplate* que já está acostumado, perfeito. Você ainda pode utilizá-lo, e eu recomendo que utilize. Mas, mesmo que você já tenha um *boilerplate* no GitHub, talvez fazer isso com IA faça sentido. Por quê? Porque essa skill pode **flexibilizar e personalizar a experiência** no processo de geração. Talvez faça sentido trabalhar de forma híbrida.

Para ficar claro: tenho dois mundos. Um mundo onde dou o comando de geração, como `create-next-app`, que gera uma estrutura. Mas, como não fico contente com isso, o que minha empresa ou eu faço? Agora, não fico mais dando esse comando. Dou um `git clone` do meu projeto base, do meu *template*. Quando faço isso, tenho todos os arquivos customizados da forma como gostaria. Acredito que você já deva trabalhar dessa forma.

Porém, mesmo trabalhando da forma mais customizada possível, ainda podem existir mudanças no projeto. Dependendo do projeto, você quer autenticação, um ORM, ou não quer dessa forma. Dependendo, quer testar de um jeito. Se é um projeto *back-end*, não preciso de testes de UI; se é um projeto *front-end*, vou precisar. Ainda assim, tenho variações e posso utilizar uma skill.

Uma skill pode fazer o seguinte: ela pode utilizar o `git clone`, ou gerar um `create-next-app`, mas ela vai chamar e usar esses comandos e, **ao mesmo tempo, fazer toda a parte de customização do projeto** para ele sair mais organizado da forma como você quer e precisa. Além disso, ela cria aquela base para que a IA consiga trabalhar.

Essa skill vai fazer com que eu tenha tudo customizado de forma muito mais estruturada, como se fosse um *tutorial* ou um *wizard*, para eu sair criando da forma como quero. E, além disso, ela já traz a aplicação pronta para uso, trabalhando com IA.
